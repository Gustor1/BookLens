# Skill: Write Specs

## Objective
Transformer l’idée utilisateur en spécification technique claire, simple et validable.

## Rules
- Sauvegarder la sortie dans `production_artifacts/Technical_Specification.md`.
- Toujours demander une validation utilisateur.
- Si l’utilisateur commente ou demande des changements, relire la spec et la réécrire.

## Instructions
1. Analyser l’idée utilisateur.
2. Rédiger une spécification avec :
- Résumé du projet
- Objectifs fonctionnels
- Contraintes techniques
- Architecture proposée
- Stack proposée
- Structure des dossiers
- Étapes de réalisation
3. Sauvegarder la spec.
4. Demander explicitement : "Approuves-tu cette spécification ?"
5. Attendre la réponse avant de continuer.
